# Creating the GitHub repository

This folder is a complete, ready-to-publish repository. Run `bootstrap_repo.sh`
from inside it, or follow the manual steps.

## Option A — GitHub CLI (`gh`), one shot

```bash
cd CSL
./bootstrap_repo.sh            # inits git, commits, sets branch 'main'
gh repo create CSL --public --source=. --remote=origin --push
```

## Option B — manual (web + git)

1. Create an empty repository named **CSL** on GitHub (no README/license — this
   folder already has them).
2. From inside this folder:

```bash
cd CSL
./bootstrap_repo.sh
git remote add origin https://github.com/<owner>/CSL.git
git push -u origin main
```

## After the first push
- Replace every `<owner>` placeholder with your GitHub username/org in
  `README.md`, `CITATION.cff`, `pyproject.toml`, and `paper/endogenous_credibility.tex`
  (the "Code and data availability" section).
- Complete the author `family-names` in `CITATION.cff`.
- Confirm the **CI** workflow (`.github/workflows/ci.yml`) goes green (it runs
  the pytest suite and a preview-scale smoke reproduction on Python 3.10–3.12).
- Tag the release and archive for a DOI (e.g. Zenodo): `git tag v1.0.0 && git push --tags`.

Simulation outputs (`results/`), caches, and LaTeX build artifacts are
`.gitignore`d by design — everything is regenerated by `make reproduce`.
